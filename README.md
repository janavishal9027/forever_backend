# Forever Backend - Spring Boot REST API

Java Spring Boot backend for the Forever e-commerce application with PostgreSQL database.

## Tech Stack

- Java 17+
- Spring Boot 3.2
- Spring Security + JWT
- Spring Data JPA / Hibernate
- PostgreSQL
- Cloudinary (image uploads)
- Stripe (payment gateway)
- Lombok

## Prerequisites

- Java 17+ (JDK)
- Maven 3.8+
- PostgreSQL running on localhost:5432
- Cloudinary account
- Stripe account (optional, for card payments)

## Setup

### 1. Create Database

```sql
CREATE DATABASE forever;
```

### 2. Configure Properties

Edit `src/main/resources/application.properties`:

```properties
# PostgreSQL
spring.datasource.url=jdbc:postgresql://localhost:5432/forever
spring.datasource.username=postgres
spring.datasource.password=YOUR_PASSWORD

# Cloudinary
cloudinary.cloud-name=YOUR_CLOUD_NAME
cloudinary.api-key=YOUR_API_KEY
cloudinary.api-secret=YOUR_API_SECRET

# Stripe
stripe.secret-key=YOUR_STRIPE_SECRET_KEY

# Admin credentials
app.admin.email=admin@forever.com
app.admin.password=admin123
```

### 3. Run

```bash
mvn spring-boot:run
```

Server starts at http://localhost:4000

Hibernate auto-creates all tables on first run (`ddl-auto=update`).

## API Endpoints

### Authentication
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/user/register` | No | Register user |
| POST | `/api/user/login` | No | User login → returns JWT |
| POST | `/api/user/admin` | No | Admin login → returns JWT |

### Products
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/api/product/list` | No | List all products |
| POST | `/api/product/single` | No | Get product by ID |
| POST | `/api/product/add` | Admin | Add product (multipart) |
| POST | `/api/product/remove` | Admin | Delete product |

### Cart
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/cart/get` | User | Get cart items |
| POST | `/api/cart/add` | User | Add to cart |
| POST | `/api/cart/update` | User | Update quantity |

### Orders
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/order/place` | User | Place COD order |
| POST | `/api/order/stripe` | User | Place Stripe order |
| POST | `/api/order/verifyStripe` | User | Verify payment |
| POST | `/api/order/userorders` | User | User's orders |
| POST | `/api/order/list` | Admin | All orders |
| POST | `/api/order/status` | Admin | Update status |

## Authentication

- JWT token passed via `token` header (or `Authorization: Bearer <token>`)
- Admin token includes `role: ADMIN` claim
- Passwords hashed with BCrypt

## Database Tables

Auto-generated by Hibernate:

| Table | Description |
|-------|-------------|
| `users` | User accounts |
| `products` | Product catalog |
| `product_images` | Product image URLs |
| `product_sizes` | Available sizes per product |
| `cart_items` | User cart items |
| `orders` | Order records with address |
| `order_items` | Line items per order |
